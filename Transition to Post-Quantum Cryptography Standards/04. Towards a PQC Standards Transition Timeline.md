# 🛡️ บทที่ 4: Towards a PQC Standards Transition Timeline

เน้นไปที่การวางแผนและกำหนดกรอบเวลาในการเปลี่ยนผ่านจากระบบรหัสลับแบบเดิมที่เสี่ยงต่อภัยคุกคามจากควอนตัม ไปสู่มาตรฐานการเข้ารหัสลับต้านทานควอนตัม (Post-Quantum Cryptography - PQC)

---

## 📅 กรอบระยะเวลาและเป้าหมายหลัก
* **เป้าหมายปี 2035**: ตามบันทึกความมั่นคงแห่งชาติ (NSM-10) สหรัฐฯ ตั้งเป้าหมายให้ระบบดิจิทัลเปลี่ยนผ่านไปสู่ระบบรหัสลับต้านทานควอนตัมให้เสร็จสิ้นภายในปี 2035
* **ความเร่งด่วน**: องค์กรต้องเริ่มเปลี่ยนผ่านก่อนที่ภัยคุกคามจากคอมพิวเตอร์ควอนตัมจะเกิดขึ้นจริง เพื่อป้องกันการโจมตีแบบ "เก็บข้อมูลวันนี้ เพื่อถอดรหัสในอนาคต" (Harvest now, decrypt later)
* **ความยืดหยุ่น**: ระยะเวลาการย้ายระบบอาจแตกต่างกันตามประเภทการใช้งานและความซับซ้อนของโครงสร้างพื้นฐาน

---

## 🔍 คำจำกัดความสถานะของอัลกอริทึม
NIST กำหนดสถานะเพื่อการบริหารจัดการการเปลี่ยนผ่าน (อ้างอิง SP 800-131A) ดังนี้:

| สถานะ (Status) | ความหมาย |
| :--- | :--- |
| **Acceptable** | ได้รับการอนุมัติให้ใช้งานได้ตามคำแนะนำ  |
| **Deprecated** | ใช้งานได้แต่มีความเสี่ยงด้านความปลอดภัย เจ้าของข้อมูลต้องยอมรับความเสี่ยงเอง  |
| **Disallowed** | **ไม่อนุญาต** ให้ใช้เพื่อการป้องกันข้อมูลอีกต่อไป  |
| **Legacy Use** | อนุญาตให้ใช้เฉพาะกับข้อมูลที่ถูกป้องกันไว้แล้วในอดีต (เช่น การตรวจสอบลายเซ็นเก่า)  |

---

## 🛡️ หมวดหมู่ความปลอดภัยใหม่ (Post-Quantum Security Categories)
NIST ใช้เกณฑ์การจัดหมวดหมู่ 5 ระดับ โดยเทียบเคียงกับความแข็งแกร่งของอัลกอริทึมแบบสมมาตร (Symmetric) ที่มีอยู่เดิม:

| หมวดหมู่ | ตัวอย่างอัลกอริทึมเทียบเคียง | ประเภทการโจมตี  |
| :---: | :--- | :--- |
| **1** | AES-128 | Key search  |
| **2** | SHA-256 | Collision search  |
| **3** | AES-192 | Key search  |
| **4** | SHA3-384 | Collision search  |
| **5** | AES-256 | Key search  |

---

## 📋 แผนการเปลี่ยนผ่านรายประเภทอัลกอริทึม

### 1. ลายเซ็นดิจิทัล (Digital Signatures) 
* **อัลกอริทึมเดิม (ECDSA, EdDSA, RSA)**:
    * ระดับความปลอดภัย 112 bits: จะถูก **Deprecated** หลังปี 2030 และ **Disallowed** หลังปี 2035 
    * ระดับความปลอดภัย ≥ 128 bits: จะถูก **Disallowed** หลังปี 2035 
* **อัลกอริทึม PQC ที่แนะนำ**: ML-DSA (FIPS 204), SLH-DSA (FIPS 205) และกลุ่ม Stateful Hash-Based (LMS, XMSS) 
### 2. การจัดตั้งกุญแจรหัส (Key Establishment) 
* **อัลกอริทึมเดิม (DH, MQV, RSA)**:
    * ระดับความปลอดภัย 112 bits: จะถูก **Deprecated** หลังปี 2030 และ **Disallowed** หลังปี 2035 
    * ระดับความปลอดภัย ≥ 128 bits: จะถูก **Disallowed** หลังปี 2035
* **อัลกอริทึม PQC ที่แนะนำ**: ML-KEM (FIPS 203) 
* **ข้อแนะนำพิเศษ**: NIST จะให้ความสำคัญกับการย้ายระบบ Key Establishment ก่อนเพื่อป้องกันการดักเก็บข้อมูลระยะยาว 

### 3. รหัสลับแบบสมมาตร (Symmetric Cryptography) [cite: 880]
* **ความทนทาน**: มาตรฐานเดิม (AES, SHA) ยังมีความแข็งแกร่งเพียงพอและได้รับผลกระทบจากควอนตัมต่ำกว่าแบบ Public-key 
* **แผนการ**: ไม่จำเป็นต้องเปลี่ยนมาตรฐานใหม่ แต่ควรหลีกเลี่ยงระดับความปลอดภัย 112 bits ซึ่งจะถูกยกเลิกในปี 2030

## 🏗️ การปรับปรุงมาตรฐานรายแอปพลิเคชัน
NIST จะดำเนินการปรับปรุงคำแนะนำเฉพาะทาง (Application-Specific) ในอนาคต ดังนี้: 
* **ลำดับความสำคัญ**: เน้นการย้ายสู่ PQC ในโปรโตคอลโต้ตอบ เช่น TLS และ IKE 
* **การประสานงาน**: ร่วมมือกับองค์กรกำหนดมาตรฐานและภาคอุตสาหกรรมเพื่อให้การเปลี่ยนผ่านเป็นไปอย่างราบรื่น
