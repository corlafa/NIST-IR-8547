# 📘 บทที่ 1: บทนำ (Introduction)

> "การเริ่มต้นเปลี่ยนผ่านสู่รหัสลับยุคหลังควอนตัมในวันนี้ คือการรับประกันความปลอดภัยของข้อมูลในอนาคต" 

อัลกอริทึมรหัสลับคือหัวใจสำคัญในการปกป้องข้อมูลอิเล็กทรอนิกส์จากการเข้าถึงโดยไม่ได้รับอนุญาต แม้ในอดีตจะมีความแข็งแกร่งเพียงพอ แต่คอมพิวเตอร์ควอนตัมในอนาคตกำลังจะทำให้มาตรการเหล่านี้ล้มเหลว 

### ⚠️ ความท้าทายที่เร่งด่วน
* **รหัสลับยุคหลังควอนตัม (PQC)**: NIST ได้เปิดตัว 3 มาตรฐานใหม่เพื่อรับมือกับภัยคุกคามควอนตัม ได้แก่ **FIPS 203 (ML-KEM)**, **FIPS 204 (ML-DSA)** และ **FIPS 205 (SLH-DSA)** 
* **ภัยคุกคาม "Harvest Now, Decrypt Later"**: ผู้โจมตีสามารถดักเก็บข้อมูลที่เข้ารหัสไว้ในปัจจุบัน เพื่อรอถอดรหัสเมื่อคอมพิวเตอร์ควอนตัมพัฒนาถึงขั้นใช้งานได้จริง 

---

### 🎯 1.1 ขอบเขตและวัตถุประสงค์ (Scope and Purpose)

การเปลี่ยนผ่านครั้งนี้ถือเป็นครั้งประวัติศาสตร์ที่มีขนาดใหญ่โตกว่าการเปลี่ยนแปลงรหัสลับครั้งใดในอดีต โดยรายงานฉบับนี้มีวัตถุประสงค์เพื่อเป็น **ก้าวแรกของกลยุทธ์ระดับกว้าง** ในการจัดการและชี้แนะแนวทางดังนี้: 

*  **การนำอัลกอริทึม PQC มาใช้**: สนับสนุนการเปลี่ยนผ่านสู่มาตรฐานใหม่ที่มีความต้านทานควอนตัม
*  **การจัดการอัลกอริทึมเดิม**: ครอบคลุมถึงการลดความสำคัญ (Deprecation), การควบคุมการใช้แบบมรดก (Legacy use) และการยกเลิกอัลกอริทึมที่เสี่ยงภัยในที่สุด 
*  **การสร้าง Roadmap**: สานต่อความร่วมมือกับภาคอุตสาหกรรมและหน่วยงานต่างๆ เพื่อสร้างกรอบเวลาที่ชัดเจนและสมเหตุสมผล 
NIST มุ่งมั่นที่จะทำให้การเปลี่ยนผ่านนี้ราบรื่นที่สุด โดยรักษาความสมดุลระหว่าง **"ความเร่งด่วนในการป้องกันข้อมูล"** และ **"การลดผลกระทบต่อระบบงาน"**

### 👥 1.2 กลุ่มเป้าหมาย (Audience)

เอกสารฉบับนี้ออกแบบมาเพื่อให้ข้อมูลแก่ผู้มีส่วนได้ส่วนเสียหลากหลายกลุ่มที่มีบทบาทในการปกป้องโครงสร้างพื้นฐานดิจิทัล: 

| กลุ่มเป้าหมาย | บทบาทในการเปลี่ยนผ่าน |
| :--- | :--- |
| **หน่วยงานรัฐบาล** | วางแผนและดำเนินงานย้ายระบบตามข้อกำหนดความมั่นคง  |
| **ผู้ให้บริการเทคโนโลยี** | พัฒนาและอัปเดตผลิตภัณฑ์ให้รองรับวิธีการรหัสลับแบบใหม่  |
| **องค์กรกำหนดมาตรฐาน** | พัฒนามาตรฐานและระเบียบวิธีที่จำเป็นสำหรับโลกยุคควอนตัม  |
| **ห้องปฏิบัติการ CMVP** | ตรวจสอบโมดูลรหัสลับให้สอดคล้องกับเกณฑ์มาตรฐานใหม่  |
