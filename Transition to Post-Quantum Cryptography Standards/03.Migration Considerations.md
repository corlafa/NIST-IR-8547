# 🛡️ สรุปแผนการเปลี่ยนผ่านสู่ Post-Quantum Cryptography (PQC)

เนื้อหาส่วนนี้สรุปความจำเป็นและแนวทางการย้ายระบบจากอัลกอริทึมเดิมที่เสี่ยงต่อคอมพิวเตอร์ควอนตัม ไปสู่อัลกอริทึมใหม่ที่ทนทานต่อควอนตัม (PQC) 

---

## 1. ข้อควรพิจารณาในการเปลี่ยนผ่าน (Migration Considerations)
แม้ปัจจุบันจะยังไม่มีคอมพิวเตอร์ควอนตัมที่ใช้งานได้จริงในระดับที่เจาะรหัสลับได้ แต่กระบวนการเปลี่ยนผ่านมักใช้เวลานานกว่า 10-20 ปี 

* **ทฤษฎีของ Mosca (Mosca’s Theorem):** องค์กรต้องเริ่มวางแผนเปลี่ยนผ่านก่อนที่ "ระยะเวลาความปลอดภัยของข้อมูลที่ต้องการ" (X) บวกกับ "เวลาที่ใช้เปลี่ยนระบบ" (Y) จะเกินกว่า "เวลาที่คาดว่าคอมพิวเตอร์ควอนตัมจะถูกสร้างเสร็จ" (Z) 
* **ภัยคุกคาม "Harvest Now, Decrypt Later":** ศัตรูอาจดักเก็บข้อมูลที่เข้ารหัสในวันนี้ เพื่อรอถอดรหัสในอนาคตเมื่อเทคโนโลยีควอนตัมพร้อมใช้งาน 
* **ความคุ้มครองข้อมูลระยะยาว:** ข้อมูลที่มีความอ่อนไหวสูง เช่น ความลับทางราชการหรือประวัติการแพทย์ จำเป็นต้องได้รับการป้องกันด้วย PQC ตั้งแต่ตอนนี้ 

---

## 2. กรณีการใช้งานหลัก (Key Use Cases) 

### 🖋️ การลงลายมือชื่อดิจิทัล (Code Signing)
* ใช้ยืนยันตัวตนผู้เขียนและรับรองว่าซอฟต์แวร์ไม่ถูกแก้ไข 
* **ความเสี่ยง:** อุปกรณ์ที่ติดตั้งโค้ดไปแล้วแต่อัปเดตไม่ได้ (เช่น อุปกรณ์ฝังตัว) ต้องถูกออกแบบให้รองรับลายเซ็น PQC ตั้งแต่ขั้นตอนผลิต 

### 👤 การยืนยันตัวตน (User & Machine Authentication)
* ระบบยืนยันตัวตนจะยังปลอดภัยตราบเท่าที่อัลกอริทึมที่ใช้ "ณ ขณะนั้น" ยังไม่ถูกเจาะ 
* **ต่างจากการเข้ารหัส:** การยืนยันตัวตนไม่มีความเสี่ยงเรื่องการดักเก็บข้อมูลไปถอดรหัสในภายหลัง (Harvest Now, Decrypt Later) 
* การเปลี่ยนผ่านต้องครอบคลุมทั้งระบบเซิร์ฟเวอร์ และอุปกรณ์ฮาร์ดแวร์โทเค็นที่ผู้ใช้ถือ 

### 🌐 โปรโตคอลความปลอดภัยเครือข่าย (Network Security Protocols)
* โปรโตคอลอย่าง **TLS, VPN หรือ IPsec** มักแยกคีย์สำหรับการแลกเปลี่ยน (Key Establishment) และการยืนยันตัวตน (Authentication) 
* **ลำดับความสำคัญ:** ควรเปลี่ยนส่วนการแลกเปลี่ยนคีย์เป็นลำดับแรก เพื่อรักษาความลับของข้อมูลจากการดักเก็บในระยะยาว 

### 📧 อีเมลและเอกสาร (Email & Document Security)
มาตรฐาน **S/MIME** ที่ใช้ RSA หรือ ECDSA เสี่ยงต่อการถูกดักเก็บข้อมูลไปถอดรหัสย้อนหลัง

## 3. โปรโตคอลแบบไฮบริด (PQC-Classical Hybrid) 
ในช่วงเปลี่ยนผ่าน NIST สนับสนุนการใช้เทคนิคไฮบริดที่ผสมผสานอัลกอริทึมเดิม (Classical) และ PQC เข้าด้วยกันเพื่อให้มั่นใจว่าระบบจะปลอดภัยหากอัลกอริทึมใดตัวหนึ่งถูกเจาะ 
| หัวข้อ | รายละเอียด |
| :--- | :--- |
| **จุดประสงค์** | เพื่อป้องกันความผิดพลาดของอัลกอริทึมใหม่ (Hedge) และรักษาการทำงานร่วมกับระบบเก่า (Legacy)  |
| **ข้อเสีย** | เพิ่มความซับซ้อนในการติดตั้ง, เพิ่มค่าใช้จ่าย และอาจส่งผลต่อประสิทธิภาพ (Performance)  |
| **สถานะ** | NIST ถือว่าเป็น **"มาตรการชั่วคราว"** เพื่อเตรียมพร้อมสู่การใช้ PQC ล้วนในอนาคต  |

### เทคนิคที่ NIST ยอมรับ:
1.  **Hybrid Key-Establishment:** ผสมค่า Shared Secret จากทั้งสองระบบเข้าด้วยกัน (ตามมาตรฐาน SP 800-56C) 
2.  **Hybrid Digital Signatures:** การใช้ลายเซ็นคู่ (Dual Signatures) บนข้อความเดียวกัน ซึ่งต้องผ่านการตรวจสอบสำเร็จทั้งคู่
